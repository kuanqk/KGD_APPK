# Промпты для запуска АППК в Claude Code

## 1. Первый запуск (один раз)

Открой терминал в папке проекта и запусти Claude Code:
```bash
cd /path/to/appk
claude
```

Затем вставь этот промпт:

```
Прочитай CLAUDE.md, docs/claude.md, docs/sprints.md и docs/rules.md.

Это Django-проект АППК — автоматизация административных дел для налоговых органов Казахстана.

Твоя задача — реализовать Sprint 0:
1. Создай структуру Django-проекта согласно docs/claude.md (раздел "Структура приложения")
2. Настрой Docker Compose: сервисы web, db, redis, worker, nginx
3. Создай кастомную модель User с полем role (5 ролей из CLAUDE.md)
4. Подключи PostgreSQL, настрой settings/base.py, dev.py, prod.py
5. Настрой AuditLog middleware (логирует вход/выход)
6. Создай Makefile с командами: up, down, migrate, test, shell
7. Создай .env.example со всеми переменными

Перед кодом — покажи план файлов которые будешь создавать. Жди моего подтверждения.
```

---

## 2. Начало каждого нового Sprint

```
Прочитай CLAUDE.md.

Начинаем [S1] — Реестр дел и карточка.

Sprint 0 завершён: Django-проект запущен, User с ролями работает, Docker Compose поднимается.

Реализуй по порядку:
1. Модели AdministrativeCase и Taxpayer (поля из docs/claude.md)
2. Автогенерацию номера дела (формат АД-ГГГГ-NNNNN)
3. ListView с фильтрами (статус, регион, ответственный)
4. DetailView — карточка дела
5. QuerySet.for_user(user) — изоляция по ролям

Начни с моделей и миграций. Покажи план сначала.
```

---

## 3. Когда нужно исправить баг

```
[BUG] 
Файл: apps/cases/services.py, строка ~47
Ошибка: [вставь текст ошибки]
Контекст: [что делал пользователь когда произошло]

Не трогай другие файлы. Только этот баг.
```

---

## 4. Когда нужна новая фича в рамках спринта

```
[S2] Добавь генерацию PDF для шаблона "Извещение о явке".
Используй WeasyPrint.
Данные берёт из AdministrativeCase через get_document_context().
Сохраняет файл в MEDIA_ROOT/cases/{case_id}/documents/.
Логирует действие в AuditLog.
```

---

## 5. Ревью кода перед коммитом

```
Проверь следующие файлы перед коммитом:
[список файлов или @apps/cases/views.py]

Убедись что:
- Нет бизнес-логики во views (только в services.py)
- Все QuerySet фильтруются через for_user(user)
- Все мутации логируются в AuditLog
- Статусы через TextChoices, не хардкод строки
- Нет print(), только logging
```

---

## 6. Добавить новую модель / изменить БД

```
[DB] Добавь в модель DeliveryRecord поле:
- tracking_number: CharField(max_length=100, blank=True)
- returned_at: DateTimeField(null=True, blank=True)

Создай миграцию. Обнови admin.py. Не трогай views.
```

---

## 7. Когда контекст засорился — начать задачу заново

Введи в Claude Code:
```
/clear
```

Затем новый чистый промпт с конкретной задачей.

---

## 8. Сохранить прогресс сессии

```
Сохрани summary текущей сессии в docs/session_ДАТА.md:
- что было сделано
- какие файлы созданы/изменены  
- что осталось в этом спринте
- известные проблемы
```

---

## Полезные команды Claude Code

| Команда | Что делает |
|---------|-----------|
| `/clear` | Очистить контекст (новая задача) |
| `/compact` | Сжать контекст (длинная сессия) |
| `/memory` | Управление автопамятью |
| `@docs/sprints.md` | Подгрузить файл в контекст |
| `Shift+Tab` | Режим планирования (план без кода) |

---

## Золотые правила промптов для этого проекта

1. **Всегда начинай с шорткода** `[S3]`, `[BUG]`, `[DB]` — экономит токены
2. **Один промпт = одна задача** — не мешай фичи с багами
3. **Сначала план, потом код** — добавляй "покажи план, жди подтверждения"
4. **При баге** — давай файл + строку + текст ошибки
5. **После 2 неудачных исправлений** — `/clear` и переформулируй
6. **Ссылайся на существующий код** — "сделай по образцу @apps/cases/views.py"
